## What
**DevOps is an organizational and technical approach** that optimizes software delivery by **connecting development and operations teams** to unify the infrastructure.

Development + Operations = Unifies Infrastructure

## Why
It emphasizes **automation, observability, continuous delivery, and reliability engineering** to enable rapid, safe, and scalable deployments.

## How

## Real-World Example (Dear Lavion)

>“When I push to the develop branch, GitHub Actions builds and tests the app, packages it as a Docker image, and pushes it to GitHub Container Registry. Then Ansible connects to my dev server over SSH, pulls the image from the registry, and runs the container. This ensures consistent, automated, and repeatable deployments.”

## Explain This Diagram (Interview)

> “CI ends when the Docker image is pushed to the registry. CD begins when Ansible pulls that image and deploys it to the dev server.”

## Tie It Directly to _Your_ Setup (Very Strong)

> “In my workflow, Ansible connects to the dev server over SSH, pulls the Docker image from the registry, stops the old container, and starts the new one—so deployments are consistent and automated.”
