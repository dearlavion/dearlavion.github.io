### **What Ansible does**

## One-Sentence Interview Answer (Very Strong)

> “Ansible is a general-purpose automation tool used for configuration management, application deployment, and operational orchestration—not just server deployments.”

## Tie It Back to Your Use (Important)

> “In my case, I primarily use Ansible for deployment automation, but the same playbooks approach can manage configuration, scaling, and operational tasks.”

### **Why I use Ansible**

> “It allows me to deploy consistently without manually logging into the server, reducing human error.”

### **How I use Ansible**

> “After GitHub Actions pushes the Docker image to the registry, Ansible connects to the dev server over SSH, pulls the image, and restarts the container.”
