
### **What**

> Modifiers are keywords in Java that **change the behavior or accessibility of classes, methods, and variables**.
> 
> - **Access modifiers** control **visibility**: `public`, `protected`, `private`, and package-private (default).
>     
> - **Non-access modifiers** change **behavior**: `static`, `final`, `abstract`, `synchronized`, `volatile`, etc.
>     

---

## **✅ Access Modifiers**

**What:**

> Control **visibility** of classes, methods, and variables.
> 
> - `public` → accessible everywhere
>     
> - `private` → accessible only inside the class
>     
> - `protected` → accessible in subclasses and same package
>     
> - default (no keyword) → accessible only in the same package
>     

**Why:**

> To **protect data**, enforce encapsulation, and control which parts of code can access members.


## **✅  Non-Access Modifiers in Java**

### **What**

> Non-access modifiers **change the behavior or properties** of classes, methods, or variables without affecting visibility.  
> Examples: `static`, `final`, `abstract`, `synchronized`, `volatile`, `transient`, `native`.

---

### **Why**

> They allow you to:

- Define **shared data** (`static`)
    
- Create **constants** (`final`)
    
- Specify **abstract behavior** (`abstract`)
    
- Ensure **thread safety** (`synchronized`, `volatile`)
    
- Optimize or interact with JVM (`transient`, `native`)
    

In short: they **control functionality, not access**.

### **How**

- **Static** → belongs to the class, not instances
    

`class Counter {     static int count = 0;  // Shared among all objects }`

- **Final** → constant value, cannot be changed
    

`final double PI = 3.1416;`

- **Abstract** → defines a method/class without implementation
    

`abstract class Shape {     abstract void draw(); }`

- **Synchronized** → thread-safe method
    

`synchronized void increment() { count++; }`
