
“In Java, if you override `equals()` to compare object content, you must also override `hashCode()`. This ensures that **hash-based collections** like `HashMap` and `HashSet` can locate objects correctly. Objects that are equal must have the same hash code.”

## **1️⃣ What is `hashCode()`**

> `hashCode()` is a method in Java that returns an **integer representing the object’s memory-based hash**.
> - Used by **hash-based collections** like `HashMap`, `HashSet`, `HashTable`.
> - Helps locate objects **quickly** in hash tables.

`Person p = new Person("Alice"); System.out.println(p.hashCode()); // some integer`

## **2️⃣ Why `hashCode()` matters**

- If you **override `equals()`** but don’t override `hashCode()`, **hash-based collections break**.

Example:
```
class Person {
    String name;
    Person(String name) { this.name = name; }

    @Override
    public boolean equals(Object obj) {
        if(this == obj) return true;
        if(obj == null || getClass() != obj.getClass()) return false;
        return name.equals(((Person)obj).name);
    }
}

Person p1 = new Person("Alice");
Person p2 = new Person("Alice");

HashSet<Person> set = new HashSet<>();
set.add(p1);
System.out.println(set.contains(p2)); // ❌ false if hashCode not overridden
```
**Why?**
- `HashSet` first checks the **hash code** to find the “bucket”
- Then calls `equals()` to check equality
- If `hashCode()` is not overridden, **p1 and p2 may have different hashes**, so `contains` fails, even though `equals()` returns true

## **3️⃣ How to fix it**

- Always override **both `equals()` and `hashCode()`** when defining logical equality
```
@Override
public int hashCode() {
    return name.hashCode();  // hash based on relevant fields
}
```
- Now `HashSet` and `HashMap` work correctly:
```
HashSet<Person> set = new HashSet<>();
set.add(p1);
System.out.println(set.contains(p2)); // ✅ true
```
