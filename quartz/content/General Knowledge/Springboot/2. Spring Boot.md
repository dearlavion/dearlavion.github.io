## What is **Spring Boot**?
**Short interview answer:**

> “Spring Boot is a framework built on top of Spring that simplifies application setup and development by removing manual configuration.”

### Why use Spring Boot?
- Auto-configuration
- Embedded servers (Tomcat, Jetty)
- Starter dependencies
- Opinionated defaults

## Why Spring Boot Exists
**Interview line:**

> “Spring Boot exists to reduce configuration overhead and allow developers to focus on business logic instead of setup.”

### ❌ Traditional Spring
- Complex XML / Java config
	In **old Spring**, you had to manually define almost everything.
	Example (Traditional Spring – XML)
```
<bean id="dataSource"
      class="org.apache.commons.dbcp.BasicDataSource">
    <property name="driverClassName" value="com.mysql.cj.jdbc.Driver"/>
    <property name="url" value="jdbc:mysql://localhost:3306/app"/>
    <property name="username" value="root"/>
    <property name="password" value="root"/>
</bean>	
```

- Manual server setup
	- You had to install **Tomcat**
	- Configure ports
	- Deploy WAR files

- Long startup time

- Steep learning curve

### ✅ Spring Boot
- Zero/low configuration
	Spring Boot uses **auto-configuration**.
- Run as a **standalone JAR**
	Spring Boot apps don’t need an external server.
	Why? Embedded Tomcat / Jetty / Undertow
- Faster development
	Spring Boot includes:
	- Starter dependencies
    - Auto-configuration
    - DevTools (hot reload)
    
	Example dependency library: spring-boot-starter-web
	Gives you:
		Spring MVC
	    Embedded Tomcat
		Jackson
		Validation
    ➡️ One dependency instead of 10+ 

- Production-ready features
	Spring Boot includes **Actuator**.
	Gives endpoints like:
		/actuator/health 
		/actuator/metrics 
		/actuator/info
	Used for:
	- Health checks
	- Monitoring
    - Kubernetes readiness/liveness probes
