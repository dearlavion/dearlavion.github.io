## Configurations
“Spring Boot supports externalized configuration via properties or YAML files. We can inject values using `@Value` for single properties or `@ConfigurationProperties` for groups. Profiles let us define environment-specific configurations, and beans can be activated conditionally using `@Profile`.”


Both files achieve the same result; YAML is preferred for complex or nested configs.

| Feature     | `application.properties` | `application.yml`                       |
| ----------- | ------------------------ | --------------------------------------- |
| Format      | Key = Value              | YAML hierarchy (key: value)             |
| Readability | Flat                     | Hierarchical, better for nested configs |
| Example     | `server.port=8080`       | `server: port: 8080`                    |

| Annotation                 | Purpose                                       |
| -------------------------- | --------------------------------------------- |
| `@Value`                   | Inject single property values                 |
| `@ConfigurationProperties` | Bind **groups of properties** into a class    |
| `@Profile`                 | Activate beans conditionally based on profile |

## Profiles

**Concept:**  
Profiles allow **environment-specific configurations** (dev, test, prod).

**Set active profile:**
```
spring.profiles.active=dev
```
**Profile-specific files:**
```
application-dev.properties
application-prod.properties
```
**Conditional beans by profile:**
```
@Profile("dev")
@Bean
public DataSource devDataSource() { return new H2DataSource(); }

@Profile("prod")
@Bean
public DataSource prodDataSource() { return new MySQLDataSource(); }
```
> “Profiles help us run the same codebase in multiple environments safely.”
