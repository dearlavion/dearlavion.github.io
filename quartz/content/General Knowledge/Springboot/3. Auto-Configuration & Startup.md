### **What is a conditional bean?**
A **conditional bean** is a bean that **only gets created if certain conditions are met**.

Spring provides annotations like:
- `@ConditionalOnClass` â†’ Create bean if a specific class is on the classpath
- `@ConditionalOnBean` â†’ Create bean if another bean exists
- `@ConditionalOnMissingBean` â†’ Only create bean if it doesnâ€™t already exist
- `@ConditionalOnProperty` â†’ Create bean based on a configuration property

#### Why is this useful?
- Spring Boot has **lots of auto-configuration** beans.
- Not all should always be created.
- Conditional beans ensure **only the necessary beans are created**, depending on:
    - Your dependencies
    - Your configuration
    - Your environment

### **Stereotype annotations**

These are **stereotype annotations**: @Service / @Component / @Repository

- Placed **on a class**. 
- Indicate: _â€œThis class is a Spring-managed bean.â€    / "Marks class as Spring Bean"
- Spring **automatically scans** your packages for these (`@ComponentScan`) and registers them in the IoC container.
Example:
```

    public void pay() { ... }

```

### **Bean**
- Placed **on a method inside a `@Configuration` class**.
- Allows **programmatic control over bean creation**.
- You can instantiate objects **manually**, or configure 3rd-party classes that you **cannot annotate**.
Example:
```
@Configuration
public class AppConfig {

    @Bean
    public RestTemplate restTemplate() {
        return new RestTemplate();
    }
}
```
